Gannet

Built with R Markdown and RStudio


Overview

Gannet is designed as a one-stop-shop MATLAB toolkit for analyzing edited 1H magnetic resonance spectroscopy (MRS) data. Its largely automated functions include the critical steps of loading raw data, substantial pre-processing, signal modeling, voxel co-registration with anatomical images, and concentration estimation based on tissue composition.

Many existing software solutions require substantial user input or offer a wide selection of processing parameters. In contrast, the purpose of Gannet is to provide users an automated pipeline with the lowest possible degree of operator input, while presenting enough useful information to judge whether the MRS experiment has been performed successfully.

Visit our website for the latest news on Gannet and our developments in edited MRS methodology.


Installation


Prerequisites

Gannet runs in MATLAB. For best performance, we recommend using the latest release if possible as Gannet is currently developed in version R2019b. Additionally, Gannet requires that the following MATLAB toolboxes are installed:

  • Image Processing
  • Optimization
  • Signal Processing
  • Statistics and Machine Learning

To run the voxel co-registration and structural image segmentation modules, SPM12 must be installed.


Installing

The simplest way to install Gannet 3.1 is to download the zipped master folder, unzip it, and then move the Gannet3.1-master folder to your MATLAB directory. To add the folder to your search path, run the following line of code in the command window of MATLAB:

macOS

addpath(genpath('~/Documents/MATLAB/Gannet3.1-master'));

Alternatively, open the Set Path dialog box from the MATLAB menu, click ‘Add with Subfolders’, find the Gannet3.1-master folder and then select it. When done, press ‘Save’ to permanently save the Gannet folder in MATLAB’s search path.

Versioning

Semantic versioning is used when updates are made to Gannet using the style 3.1.x. Versioning is also conducted on a module-specific basis using the style YYMMDD. That is, each Gannet module has its own release version.
